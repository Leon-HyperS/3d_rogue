## Recommended Runtime Format

When building a web game in this repo:

- use `glTF` first
- treat `FBX`, `OBJ`, and `.blend` as source/reference formats only unless there is a strong reason otherwise

## Recommended Curation Approach

Do not point runtime loading at the full `Free_Assets/` tree if you can avoid it.

Instead:

1. choose a small game concept
2. identify one player set, one environment set, one pickup set, and one enemy set
3. copy only the chosen runtime assets into `public/assets/`
4. create `public/assets/assets_index.json`
5. load from the curated runtime area, not the full source library

## Suggested Starter Pairings

### 3D Dungeon Stealth Prototype

Suggested source pairing:

- player: `Platformer Game Kit - Dec 2021/Character/glTF/Character.gltf`
- enemies: `Ultimate Monsters/Big/glTF/Orc.gltf`, `Demon.gltf`, `Yeti.gltf`
- pickups: `Powerups and Pickups/glTF/Key.gltf`
- goal: `Level and Mechanics/glTF/Chest.gltf`
- gate/interactions: `Door.gltf`, `Lever.gltf`, `Fence_*`

### 3D Arena Survival Prototype

Suggested source pairing:

- player: `Character_Gun.gltf`
- enemies: `Ultimate Monsters/Flying/glTF/*` or `Big/glTF/*`
- pickups: `Coin.gltf`, `Heart.gltf`, `Thunder.gltf`
- hazards: `Spikes.gltf`, `Hazard_Saw.gltf`, `Bomb.gltf`

### 3D Platformer Prototype

Suggested source pairing:

- player: `Character.gltf`
- enemies: `Crab.gltf`, `Bee.gltf`, `Skull.gltf`
- terrain: `Modular Platforms/3D/glTF/*`
- environment: `Nature/glTF/*`
- goals and rewards: `Goal_Flag.gltf`, `Coin.gltf`, `Gem_*`, `Heart.gltf`

## Current Recommendation

Until a specific game concept is chosen:

- keep `Free_Assets/` as the source library
- do not duplicate everything into runtime folders
- curate only the subset needed for the chosen game
